# 공용 화면 요소 작업 규칙

## 담당 범위

이 디렉터리는 버튼·키패드·문제 카드·피드백·결과·진도, 학년별 미션 표현, SVG 문제 그림, 풀이장과 보상 표현을 담당한다. 문제 선택, 정답 계산, 세션 만료, 진도 보상 정책은 담당하지 않는다.

## 경계

- 컴포넌트는 받은 문제와 공개 상태를 표현한다. 정답을 다시 계산하거나 localStorage에 새 영속 형식을 쓰지 않는다.
- 학년 전용 표현은 `grade1`, `grade2`, `grade3` 아래에 유지한다. 실제로 둘 이상의 학년에서 같은 상태 의미와 상호작용을 쓸 때만 공용으로 올린다.
- 시각 컴포넌트가 새 데이터 필드를 필요로 하면 타입·생성기·검증기와 함께 계약을 바꾼다. 선택적으로 필드가 있다고 가정해 조용히 잘못 그리지 않는다.
- 미션·캐릭터·보상 표현은 답안 저장·채점 성공 여부를 통제하지 않는다.

## 항상 유지할 동작

- `result`, `target`, `product`, 정답 치수와 같은 답 전용 값은 확인 또는 풀이 공개 전 DOM과 접근성 이름에 존재하지 않아야 한다.
- 문제 자체가 주는 개수·길이·분류 표식은 숨기지 않는다. 문제 데이터와 정답 데이터를 필드 이름만으로 일괄 마스킹하지 않는다.
- 정량 SVG는 문제와 같은 수학 모델에서 위상·비율·라벨을 만들고 0인 영역을 그리지 않는다. 고정 좌표를 눈으로만 맞춰 정량 그림으로 배포하지 않는다.
- 터치 가능한 주요 요소는 최소 48×48px이며 한 화면의 중심 행동이 경쟁하는 여러 강조 버튼으로 갈라지지 않아야 한다.
- 숫자 입력은 정수·소수·분수·대분수에 필요한 `0-9`, `.`, `/`, 공백, 음수, 지우기, 완료를 안정적으로 제공한다.

## 풀이장 구현 계약

- 한 번에 하나의 활성 `pointerId`만 그리며 보조 포인터와 마우스 보조 버튼은 무시한다.
- 시작점을 즉시 그려 짧은 점 입력을 보존하고 coalesced event로 빠른 이동을 순서대로 그린다.
- 명시적 포인터 캡처 실패가 스트로크를 버리게 하지 않으며 전역 종료 이벤트를 안전망으로 둔다.
- 연습 상호작용 영역 전체의 WebKit 텍스트 선택과 콜아웃을 막고 `input`, `textarea`만 선택을 허용한다.
- 캔버스 pointer·touch 시작과 이동 네이티브 리스너는 캡처 단계의 non-passive여야 한다.

## 테스트

- 시각 공개 상태는 제출 전 부재와 제출 후 존재를 한 쌍으로 검사한다.
- `ScratchPad.test.ts`는 선택 방지, 단일 포인터, 시작점, 캡처 실패를 보호해야 한다. WebKit 변경은 계산 스타일·선택 문자열·취소 이벤트·그려진 픽셀을 실제 엔진에서 추가 확인한다.
- 정량 그림은 렌더 문자열 존재만 보지 말고 위상, 비율 오차, 라벨 내부 위치, 0 영역 비표시를 검사한다.
- 공용 컴포넌트 변경은 영향받는 학년의 브라우저 흐름과 모바일·태블릿 레이아웃을 함께 확인한다.
