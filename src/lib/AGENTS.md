# 학습 코어 작업 규칙

## 담당 범위

이 디렉터리는 수학 함수, 문제 생성, 답 정규화·채점, 학년별 문제 은행, 세션·결과·진도 저장 정규화, 보상 투영, 홈 요약과 공유 타입을 담당한다. 경로 레이아웃, SVG 좌표, 버튼 문구는 담당하지 않는다.

## 경계

- 이 디렉터리는 React 화면 컴포넌트를 가져오지 않는다.
- 정답과 채점은 등록된 수학 함수와 명시적 정규화로 계산한다. 원격 AI나 자연어 의미 추정에 의존하지 않는다.
- `types.ts`, `problem-generator.ts`, `session.ts`는 여러 학년과 화면이 공유한다. 변경 전 공유 작업 기록에 새 필드, 이전 형식, 소비자를 명시한다.
- 저장 로더는 브라우저가 없는 환경에서 안전한 기본값을 반환하고, 손상된 한 키 때문에 다른 키를 지우지 않는다.
- 문제 은행은 화면 문구·스타일을 직접 소유하지 않지만 답 형식과 시각 데이터가 화면에서 검증 가능하도록 완전한 계약을 제공한다.

## 항상 유지할 동작

- 같은 템플릿·매개변수·시드는 같은 문제 문장, 정답, 풀이와 시각 데이터를 만든다.
- 객관식 보기는 네 개가 모두 다르고 정답이 정확히 하나다. 셔플 뒤 정답 위치는 정답 값과 일치해야 한다.
- 숫자 답은 정수·소수·분수·대분수의 수학값으로 비교한다. 분모 0과 부분 파싱은 거부한다.
- 정규화 실패는 채점 가능한 오답이 아니라 입력 형식 실패로 호출자에게 구분되어야 한다. 완성되지 않은 형식이 세션 확인 상태를 바꾸면 안 된다.
- 5학년 세션은 2시간 후 만료하고, 확인한 답은 `updateAnswer`로 바뀌지 않는다. 오답 복습은 직전 결과의 틀린 문제만 포함한다.
- 1·2학년 중복 보상 키는 미션 ID와 렌더된 문장·정답·보기·시각 내용을 반영한다. 시드만 달라진 동일 문제에 새 보상을 주지 않는다.
- 일일 시드, 오늘 해결 수, 학습일과 연속 학습은 기기의 같은 현지 날짜 경계를 사용한다.
- 이전 진도 형식을 읽을 때 완료·복습·선택 단원·최근 활동을 보존한다. 홈 요약은 원본 학년 키를 다시 쓰지 않는다.

## 구현 방식

- 템플릿 표현식은 허용된 변수·제한 산술·등록 함수만 평가하고 알 수 없는 식을 성공처럼 처리하지 않는다.
- 새 답 형식은 파싱, 정규화, 비교, 표시를 분리하고 형식 오류와 수학적 오답을 구분한다.
- 새 저장 형식은 버전을 올리고 이전 예제에서 새 형식으로 읽는 집중 테스트를 먼저 만든다.
- 공통 학습 활동을 도입할 때 기존 학년별 저장 이름을 즉시 제거하지 않고 읽기 어댑터로 유지한다.

## 테스트

- 수학 경계값, 잘못된 입력, 분수 동치, 중복 보기, 시드 재현, 세션 만료, 손상 JSON, 이전 형식 복구를 집중 테스트한다.
- 생성기나 문제 은행 변경은 장기 시드 표본에서 구체 문제 다양성과 중복 보상 지문을 검사한다.
- 타입이나 시각 계약 변경은 렌더러 테스트와 콘텐츠 검증기를 같은 변경에서 갱신한다.
- 제품 파일 변경은 `.codex/hooks/tdd-guard.sh`가 집중 테스트 변경을 인식해야 한다.
