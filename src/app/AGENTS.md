# 화면 경로 작업 규칙

## 담당 범위

이 디렉터리는 공개 경로, 빌드 시 정적 식별자 생성, 화면별 클라이언트 상태 조립, 사용자 행동과 이동을 담당한다. 정답 공식, 숫자 정규화, 문제 은행, 영속 형식의 복구 규칙은 이 디렉터리의 범위가 아니다.

## 경계

- 화면은 문제 생성·채점·진도 계산을 `src/lib`의 공개 함수로 요청한다. 페이지 안에 같은 규칙을 다시 만들지 않는다.
- 재사용 가능한 입력·도형·보상 표현은 페이지 파일에 복제하지 않고 `src/components`에서 제공한다.
- 5학년 단원·개념·템플릿 내용을 페이지에 하드코딩하지 않는다. 정적 데이터 로더를 사용한다.
- 새 localStorage 이름을 화면 컴포넌트에 직접 추가하지 않는다. 기존 학년 화면의 저장 호출을 바꿀 때는 해당 진행 모듈의 이전 형식 복구를 함께 다룬다.
- `/review/problems`는 내부 검수 경로다. 랜딩과 학습 홈의 어린이용 주요 행동에 노출하지 않는다.

## 항상 유지할 동작

- `/`는 소개와 하나의 학습 시작 행동, `/home`은 기기 로컬 이어하기와 학년 선택, `/grade/5`는 5학년 단원 목록을 소유한다.
- 홈의 명시적 선택 학년이 우선이고, 선택이 없으면 가장 최근 활동 학년을 사용한다. 만료되지 않은 5학년 세션은 5학년 이어하기에서 우선한다.
- 5학년 연습은 요청한 개념·세트·모드·원본 결과가 모두 맞을 때만 기존 세션을 복구한다. 다른 요청의 세션을 잘못 이어 붙이지 않는다.
- 답을 확인한 문제는 다시 수정할 수 없고, 모든 문제를 확인하기 전에는 결과 제출이 활성화되지 않는다.
- 숫자·구조화 입력은 정규화에 성공한 뒤에만 답을 확인한다. 완성되지 않은 형식은 안내만 보여 주고 답을 잠그거나 오답으로 기록하지 않는다.
- 학년 하나의 손상된 진행 기록 때문에 다른 학년 기록을 초기화하지 않는다.
- 모든 내부 링크와 정적 자산은 `/math_assist` 기준 배포에서도 작동해야 한다.

## 구현 방식

- 정적 식별자 경로는 빌드 시 데이터 파일에서 `generateStaticParams`로 생성한다.
- localStorage를 읽는 화면은 서버 렌더 결과가 아니라 hydration 뒤 상태를 기준으로 복구 UI를 확정한다.
- 2·3학년 미션의 잘못된 `unitId`는 각 학년 첫 유효 단원으로 제한한다.
- 연습 실패와 결과 부재는 빈 화면이나 무한 로딩 대신 어린이가 이해할 수 있는 안내와 홈 이동을 제공한다.

## 테스트

- 경로, 이동, 즉시 확인, 오답 복습, 결과 행동이 바뀌면 `e2e/learning-loop.spec.ts`를 갱신한다.
- localStorage 선택·복구 로직은 화면 테스트보다 `src/lib`의 집중 테스트에서 입력 형식을 고정하고, 브라우저 테스트로 실제 연결을 확인한다.
- 모바일 390×844와 태블릿 1024×768에서 가로 스크롤, 고정 하단 행동 가림, 터치 목표를 확인한다.
- 개발 서버와 프로덕션 빌드를 동시에 실행하지 않는다.
