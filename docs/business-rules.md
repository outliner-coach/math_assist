# 학습 규칙

## 학습자와 범위

- 현재 작업트리 제공 학년은 1·2·3·4·5·6학년이다. 4학년은 검증된 큰 수 단원, 6학년은 검증된 비와 비율 단원만 제공한다. 실제 GitHub Pages 반영은 별도 배포 증거가 필요하다.
- 학습 시작에는 계정이 필요하지 않다. 기록을 여러 기기에 보존하려는 학습자만 `기록 저장하기`를 선택한다.
- 주 사용자는 초등학생이며 보호자는 보조 사용자다. 어린이가 이해할 수 없는 계정 절차를 학습 진입 조건으로 두면 안 된다.

## 정답과 채점

- 정답 계산과 채점은 같은 입력에 항상 같은 결과를 내는 규칙으로 수행한다. 자연어 생성 결과나 캐릭터·보상 상태가 정답을 바꾸면 안 된다.
- 채점 결과는 정답 또는 오답뿐이며 부분 점수는 없다. 숫자·구조화 입력의 형식이 잘못된 상태는 수학적 오답과 구분해 다시 입력하게 한다. `1/`, `-`, `.`처럼 완성되지 않은 입력은 답을 잠그거나 오답으로 기록하지 않고 구체적인 형식 안내를 보여 준다.
- 객관식은 선택한 보기와 정답 보기의 일치로 판정한다. 숫자 답은 앞뒤 공백을 제거한 뒤 정수·소수·분수·대분수로 정규화하여 값이 같은지 비교한다.
- `2/4`와 `1/2`는 같은 답이다. `1 1/2`는 내부적으로 `3/2`와 같은 값이다. 분모가 0이거나 숫자 형식 전체가 일치하지 않으면 유효한 답으로 처리하지 않는다.
- 2·3학년의 길이·시간·분수·들이·무게처럼 여러 칸으로 받은 답은 단위와 각 부분을 정규화한 뒤 하나의 수학값으로 비교한다. 자유 서술문을 의미 추정으로 채점하면 안 된다.

## 5학년 연습

- 표준 연습은 한 개념의 A·B·C 세트 중 하나에서 10문제를 제공한다. 각 세트의 난이도 1·2·3 구성은 4·4·2여야 한다.
- 학습자는 각 문제의 답을 확인한 직후 정오와 단계별 풀이를 본다. 확인 전에는 다음 단계의 정답 전용 정보를 보여 주지 않으며, 확인한 답은 같은 세션에서 바꾸지 않는다.
- 전체 결과는 10문제를 모두 확인한 뒤에만 완성한다. 확인되지 않은 문제가 하나라도 있으면 최종 제출을 허용하지 않는다.
- 진행 중 세션은 시작 후 2시간 동안만 복구한다. 만료한 세션은 이어서 풀지 않고 새 세션을 시작한다.
- 오답 재도전은 직전 결과에서 틀린 문제만 복사한다. 직전 결과가 없거나 요청한 결과·개념·세트가 맞지 않으면 임의의 복습 세션을 만들지 않는다.

## 6학년 Study 연습

- 아래 계약은 공개 curriculum ledger가 정확히 `released`일 때만 활성화한다. 원장 부재·fetch·schema·상태 오류에서는 unit·concept·practice·result·retry가 문제나 기존 결과를 노출하지 않고 저장을 변경하지 않는다.
- 현재 공개 범위는 `unit-6-1-ratio`의 `g6ratio-001`이며 `[6수02-02]`, `[6수02-03]`에 추적한다. `[6수02-04]` 비례식 성격 문항은 이후 별도 범위다.
- A·B·C는 각각 서로 겹치지 않는 10개 문제군과 난이도·K/A/R 4/4/2를 유지한다. 기본 세션은 5문제(2/2/1), 선택 세션은 10문제(4/4/2)다. 표 blueprint는 제출 전 정답 전용 값이 없는 실제 정량 표를 렌더링한다.
- Grade 6 세션·결과·진도는 Grade 5 저장 이름을 읽거나 쓰지 않는다. 오답 재도전과 새 문제는 원 세션의 5/10 선택을 유지한다.
- 풀이장 session ID는 `grade6_session` 접두사를 사용하고 receipt는 `grade: 6`, `grade6-ratio-v1`을 기록하되 원답과 획은 포함하지 않는다.
- 개념 진도는 시도 수, 최고·최근 정답률, 최근 완료 시각, 복습 필요 여부를 저장한다. 오답이 하나라도 있으면 복습 필요이며, 만점 결과는 오답 재도전을 제안하지 않는다.
- 손상된 Grade 6 session/result는 자동으로 지우거나 덮어쓰지 않는다. 화면에서 원문 보존을 알리고 해당 Grade 6 키만 명시적으로 초기화한 뒤 새 학습을 시작한다.

## 4학년 Bridge

- 현재 공개 범위는 `[4수01-01]`, `[4수01-02]` 큰 수 단원 하나다. 한 활동은 Knowing·Applying·Reasoning 각 1문제, 총 3문제다.
- 유효한 오답 뒤 맞힌 문제는 복습 목록에 남긴다. 형식 오류는 오답·시도 순번·receipt로 기록하지 않는다.
- 홈은 선택 단원이 있으면 `/grade/4/mission?unitId=`로 이어 주고, 손상된 4학년 기록을 읽는 동안 다른 학년 기록이나 손상 원문을 바꾸지 않는다.

## 1·2·3학년 미션과 보상

- 미션의 문제 내용과 정답은 결정적이어야 한다. 같은 날짜와 재도전 순서가 같은 구체 문제를 재현할 수 있어야 한다.
- 1·2학년은 문제 문장·정답·보기·시각 데이터가 같은 구체 문제를 다시 풀어도 경험치, 오늘 해결 수, 숙련 정답 수를 중복 지급하지 않는다. 시드만 다르고 내용이 같으면 새 보상 대상이 아니다.
- 정답인 새 구체 문제는 기본 10 경험치를 준다. 힌트 없이 첫 시도에 맞히면 5, 복습 예정일 이후 복습을 마치면 5를 더한다. 1학년 난이도 3과 2학년 적용형 미션은 난이도 보너스 5를 추가한다. 힌트를 썼다는 이유로 이미 얻은 경험치를 깎거나 기본 완료 보상을 줄이면 안 된다.
- 레벨은 100 경험치마다 하나씩 오른다. 오늘 목표는 기기 현지 날짜 기준 8문제이며, 날짜 시드·학습일·연속 학습·오늘 목표는 모두 같은 현지 자정을 경계로 바뀐다.
- 숙련 별은 새 구체 문제 정답, 힌트 없는 첫 시도 정답, 예정일 이후 복습 정답의 세 근거로 최대 3개다. 복습 간격은 1·3·7·21일 순서로 늘어난다.
- 하루를 쉬었다고 즉시 성취를 잃게 만들지 않는다. 리더보드, 손실형 연속 기록, 힌트 벌점, 콘텐츠 잠금으로 어린이를 압박하지 않는다.
- 3학년은 현재 별도의 진도 형식을 사용한다. 1·2학년의 경험치 규칙을 3학년에 자동 적용하지 않으며 공통 활동 규칙을 도입할 때 명시적으로 이관한다.

## 문제와 그림의 관계

- 그림은 장식, 관계를 보여 주는 개략도, 수치를 실제 풀이에 쓰는 정량 그림 중 하나로 분류한다.
- 모든 그림은 제출 또는 풀이 공개 전까지 정답·결과·목표값과 같은 답 전용 정보를 숨긴다. 분류용 개수 표식처럼 문제 자체가 제공하는 데이터는 숨기지 않는다.
- 정량 그림의 문제 수치·정답·영역 위상·비율·라벨은 하나의 수학 모델에서 파생한다. 문제와 무관한 고정 좌표 그림으로 정량 관계를 흉내 내면 안 된다.
- 정량 그림은 0인 영역을 보이지 않아야 하고 양수 영역은 식별 가능해야 한다. 핵심 대소 관계와 교집합 위상은 모두 보존하고, 길이·넓이 비율은 목표의 ±10% 안에 있어야 하며 라벨 중심은 해당 영역 안에 있어야 한다.

## 장기 학년군 경험

- 1·2학년은 1~3문제의 놀이형 미션, 3·4학년은 3~5문제의 연결형 미션, 5·6학년은 5문제 기본·10문제 선택의 학습형 경험을 목표로 한다.
- 학년군은 문제 수, 글 밀도, 캐릭터 크기, 풀이 도구, 보상 노출을 다르게 할 수 있지만 정답·채점·진도 의미를 서로 다르게 정의하면 안 된다.
- 캐릭터와 미션은 학습 시작·재도전·다음 행동을 돕는 표현이다. 문제 로딩, 답안 저장, 채점, 복구 성공 여부가 캐릭터 애니메이션에 의존하면 안 된다.

## 기록 저장 계정

- 어린이는 스스로 `기록 저장하기`를 선택할 수 있다. 이 선택이나 기기 로컬 저장은 보호자 절차로 막지 않지만, 원격 기록 생성은 필요한 법정대리인 동의와 동의자 확인 뒤 진행한다.
- 계정은 자동 생성된 학습 번호와 어린이가 정한 4자리 숫자 비밀번호를 사용한다. 별명은 선택 사항이며 실제 이름·이메일·휴대폰·학교 정보는 요구하지 않는다.
- 처음 만든 기기는 로그인 상태를 유지한다. 다른 기기에서는 학습 번호와 4자리 숫자 비밀번호를 모두 입력해야 한다.
- 가입 시 일회성 복구 카드를 제공한다. 복구 카드도 없고 보호자 연결도 없다면 잊어버린 비밀번호를 신원 추정으로 복구하지 않는다.
- 첫 동기화는 완료 기록, 복습 상태, 최근 활동, 선택 아바타만 포함한다. 풀이장 획, 캐릭터 표정, 팝업 상태, 지도 스크롤, 애니메이션 상태는 동기화하지 않는다.
