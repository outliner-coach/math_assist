# 로그인 없는 랜딩·학습 홈 V1

작성일: 2026-07-19
주 작업영역: Workstream 03 - UI and Visuals

## 목적

기존 `/` 화면에 섞여 있던 1·2·3학년 탐험 진입, 5학년 단원 목록,
내부 문제 검수 링크를 분리한다. 로그인 도입 전까지는 학습자를 계정이 아닌
현재 브라우저의 로컬 기록으로 인식한다.

## 라우팅

- `/`: 제품 소개와 단일 학습 시작 행동만 제공하는 랜딩
- `/home`: 학년 선택, 최근 학습 복귀, 공통 진도 요약을 제공하는 게스트 홈
- `/grade/1`, `/grade/2`, `/grade/3`: 기존 학년별 탐험 흐름
- `/grade/4`: 검증된 큰 수 Bridge 단원과 3문제 활동
- `/grade/5`: 기존 `/`에 있던 5학년 12개 단원 선택
- `/grade/6`: 검증된 비와 비율 Study 단원과 5/10문제 활동
- `/review/problems`: 직접 URL은 유지하되 학습자용 랜딩과 홈에서는 노출하지 않음

## 홈 상태 규칙

### 최초 방문

- 저장된 학년과 진행 기록이 모두 없으면 1·2·3·4·5·6학년 선택을 먼저 보여준다.
- 이름, 이메일, 비밀번호는 요구하지 않는다.

### 학년 선택 후 첫 학습 전

- 의미 없는 0% 통계 대신 첫 학습 행동과 안내 문구를 보여준다.
- 2·3학년에서 단원만 선택했다면 해당 단원의 `첫 미션 시작`으로 복귀한다.

### 진행 기록이 있을 때

- 명시적으로 고른 학년을 우선한다.
- 저장된 학년이 없으면 `lastPlayedAt`이 가장 최근인 학년을 연다.
- 1·2·3·4학년은 완료 미션 또는 variant, 복습, 오늘 해결 수를 요약한다.
- 5·6학년은 완료 개념, 복습 개념, 현재 기기 저장 상태를 요약한다.
- 만료되지 않은 5학년 10문제 또는 6학년 5/10문제 세션이 있으면 해당 학년에서 다른 기록보다 먼저 이어서 푼다.

## 저장 계약

`mathAssist_guestHome_v1`에는 현재 `activeGrade`만 저장한다. 기존 학년별
진행 키는 변경하거나 이동하지 않는다. 이 분리는 향후 계정 또는 학습자 프로필을
추가할 때 홈 UI를 유지하고 데이터 제공자만 교체하기 위한 경계다.

학습자는 홈에서 다음 사실을 확인할 수 있어야 한다.

> 학습 기록은 현재 브라우저에 저장돼요. 브라우저 데이터를 지우거나 다른
> 기기를 사용하면 기록이 이어지지 않을 수 있어요.

## 검증 기준

- 최초 랜딩의 대표 행동이 `/home`으로 이동한다.
- 최초 홈은 학년 선택을 보여준다.
- 학년 선택은 새 진행 기록을 만들거나 기존 기록을 삭제하지 않는다.
- 미션을 완료하고 홈으로 돌아오면 완료/오늘/복습 요약이 갱신된다.
- 5학년 단원과 개념의 기존 경로가 `/grade/5`를 통해 유지된다.
- 390×844 모바일과 1024×768 태블릿에서 가로 스크롤 없이 사용할 수 있다.
