# 해결되지 않은 문제

## 원격 계정의 삭제·보존 정책이 정해지지 않음

- 조건과 증상: 승인된 로그인 경험은 학습 번호, 4자리 숫자 비밀번호, 복구 카드, 선택적 보호자 연결까지 정해졌지만 계정 삭제 뒤 서버 기록 보존 기간, 보호자 연결 해제 시 권한·복구 처리, 로그인 세션 만료 시간이 없다.
- 영향: 계정 삭제가 실제 삭제인지, 연결을 끊은 보호자가 계속 복구할 수 있는지, 잃어버린 기기의 세션이 언제 끝나는지 일관되게 구현하거나 안내할 수 없다.
- 지금 해결하지 못하는 이유: 기술만으로 정할 수 없는 제품·개인정보 정책이며 사용자의 추가 결정이 필요하다.
- 가능한 접근: 계정 삭제 즉시 접근 차단과 짧은 복구 유예 후 영구 삭제, 보호자 연결 해제 즉시 복구 권한 철회, 어린이 기기의 장기 로그인과 위험 사건 발생 시 전체 세션 철회를 함께 검토한다.

## 5학년의 완성되지 않은 숫자 입력이 승인된 규칙과 다름

- 조건과 증상: 5학년 숫자 입력에서 `1/`, `-`, `.`처럼 비어 있지는 않지만 완성되지 않은 값을 확인하면 정규화가 실패해 오답이 되고, 확인한 답은 같은 세션에서 수정할 수 없다. 2·3학년 구조화 입력은 같은 종류의 형식 오류를 별도로 안내한다.
- 영향: 계산을 틀린 것이 아니라 입력을 덜 마친 어린이도 오답 기록과 잠긴 답을 받는다. 학년별 피드백 의미도 달라진다.
- 지금 해결하지 못하는 이유: 형식 오류를 오답과 분리하기로 결정했지만 현재 작업 범위는 문서 정리이며, 제품 코드를 바꾸려면 집중 테스트와 브라우저 회귀 검증을 동반해야 한다.
- 가능한 접근: 확인 전에 정규화 성공 여부를 검사하고 실패하면 답을 잠그거나 오답 횟수를 늘리지 않은 채 구체적인 형식 안내를 보여 준다.
