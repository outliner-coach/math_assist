# 해결되지 않은 문제

## 원격 계정의 삭제·보존 정책이 정해지지 않음

- 조건과 증상: 승인된 로그인 경험은 학습 번호, 4자리 숫자 비밀번호, 복구 카드, 선택적 보호자 연결까지 정해졌지만 계정 삭제 뒤 서버 기록 보존 기간, 보호자 연결 해제 시 권한·복구 처리, 로그인 세션 만료 시간이 없다.
- 영향: 계정 삭제가 실제 삭제인지, 연결을 끊은 보호자가 계속 복구할 수 있는지, 잃어버린 기기의 세션이 언제 끝나는지 일관되게 구현하거나 안내할 수 없다.
- 지금 해결하지 못하는 이유: 기술만으로 정할 수 없는 제품·개인정보 정책이며 사용자의 추가 결정이 필요하다.
- 가능한 접근: 계정 삭제 즉시 접근 차단과 짧은 복구 유예 후 영구 삭제, 보호자 연결 해제 즉시 복구 권한 철회, 어린이 기기의 장기 로그인과 위험 사건 발생 시 전체 세션 철회를 함께 검토한다.

## 2026-07-21에 해결한 문제

### 6학년 비와 비율 release-candidate 품질 미달

- 원인: A·B·C가 같은 수학 구조를 반복했고, `table` metadata가 실제 표 표현과 연결되지 않았으며, 일부 비례식 성격 문항과 어린이 문장이 현재 `[6수02-02/03]` 범위를 벗어났다.
- 해결: 세트 사이에 겹치지 않는 30개 문제군으로 다시 설계하고 `[6수02-04]` 성격의 문항을 제외했다. 세트별 추론 문항은 정답 전용 값이 없는 실제 정량 `ratio_table`을 렌더링한다. 손상된 6학년 세션·결과는 원문을 보존하면서 명시적 초기화 UI를 제공한다.
- 확인: Grade 6 validator와 품질 감사에서 6학년 경고 0, 전용 Chromium E2E 5/5로 홈 진입, 5/10문제, 실제 표, 손상 복구, Grade 5 비간섭, 390×844·1024×768와 48px 동작을 확인했다.
- 유지 계약: 공개 원장 상태가 정확히 `released`일 때만 학습 경로를 열며 원장 로드·schema·상태 오류는 계속 fail-closed한다. 이후 비례식 `[6수02-04]` 범위는 별도 단원·검토로만 추가한다.

### 5학년 템플릿 9개의 문장·단원·정답 규칙 불일치

- 원인: `fracmul`은 덧셈 문맥과 곱셈 solver가 충돌했고, `fracsub`은 매개변수 조합에 따라 음수가 될 수 있었으며, `average`는 평균 대신 합을 물었다.
- 해결: 곱셈은 전체의 일부 중 일부, 뺄셈은 처음 양에서 사용한 양, 평균은 네 값의 평균 문맥으로 문장·범위·solver·풀이·blueprint를 함께 고쳤다.
- 확인: 생성 self-check와 5학년 집중 회귀가 통과했고 청사진은 660/660, 전체 template metadata는 690/690이다.
- 유지 계약: 문장·매개변수 범위·solver·풀이·성취기준을 하나의 의미 단위로 검토하며, 의미가 맞지 않는 문항에 임시 metadata만 붙여 감사 수를 채우지 않는다.
