# 게스트 시작과 선택적 어린이 계정

## 상황

기기 로컬 기록은 가장 단순하지만 다른 기기에서 이어갈 수 없고 브라우저 데이터를 지우면 사라진다. 반대로 이메일·휴대폰 인증이나 보호자 승인을 먼저 요구하면 초등학교 1학년이 혼자 학습을 시작하기 어렵다.

## 결정

모든 학습은 게스트로 시작한다. 기록을 보존하려는 어린이는 보호자 승인 없이 `기록 저장하기`를 눌러 계정을 만들 수 있다. 계정은 자동 생성 학습 번호와 어린이가 정한 4자리 숫자 비밀번호를 사용하고, 가입 때 일회성 복구 카드를 제공한다. 별명은 선택이며 실제 이름·이메일·휴대폰·학교 정보는 요구하지 않는다.

## 선택하지 않은 안

- 이메일 또는 휴대폰 인증은 저학년이 독립적으로 완료하기 어렵고 필요 이상의 개인정보를 요구하므로 선택하지 않았다.
- 보호자 승인을 계정 생성의 필수 단계로 두는 방식은 보호자가 곁에 없을 때 기록 저장 자체를 막으므로 선택하지 않았다.
- 별명과 4자리 숫자만으로 계정을 찾는 방식은 별명 충돌과 계정 추측 위험이 커서, 서버가 생성한 학습 번호를 함께 쓰기로 했다.

## 결과

4자리 숫자는 기억하기 쉽지만 가능한 조합이 적으므로 반복 실패 지연·일시 차단과 추측하기 어려운 학습 번호가 필수다. 복구 카드와 연결된 보호자가 모두 없으면 신원 추정 복구는 불가능하다. 계정 화면, 서버 저장, 세션 철회, 삭제·보존 정책이 준비되기 전에는 이 기능을 제공할 수 없다.
