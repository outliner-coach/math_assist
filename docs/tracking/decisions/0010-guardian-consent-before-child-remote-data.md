# 아동 원격 기록 전 법정대리인 동의 확인

## 상황

게스트 학습과 기기 로컬 기록은 현재 브라우저 프로필 안에서만 동작한다. 선택적 원격 계정은 학습 번호, 완료·복습, 최근 활동, 선택 아바타를 서버에서 처리하므로 만 14세 미만 아동 개인정보 처리의 법적·운영 경계를 새로 만든다.

개인정보 보호법 제22조의2는 만 14세 미만 아동의 개인정보 처리에 법정대리인 동의가 필요한 경우 그 동의를 받고 법정대리인이 동의했는지 확인하도록 요구한다. 아동에게 고지할 때는 이해하기 쉬운 양식과 언어를 사용해야 한다.

## 결정

- 계정 없이 문제를 풀고 기기 로컬에 저장하는 흐름은 계속 즉시 제공한다.
- 어린이는 `기록 저장하기`를 직접 선택할 수 있지만, 원격 learner record와 인증 계정을 실제로 생성·동기화하기 전 법정대리인 동의와 동의자 확인을 완료한다.
- 동의 확인에 필요한 최소 정보만 별도 목적과 보존 기간으로 처리한다. 실제 이름·학교·어린이 이메일은 학습 계정의 기본 필드로 추가하지 않는다.
- 적용 가능한 적법 근거, 동의 증명, 철회, 삭제·보존, 보호자 연결 해제, 세션 만료, 아동용 고지가 개인정보 검토에서 승인되기 전 production 원격 기능 플래그는 꺼 둔다.
- 보호자 부재 때문에 학습 또는 기존 로컬 기록이 막히거나 삭제되어서는 안 된다.

## 대안과 이유

- 동의 없이 어린이 단독으로 원격 계정을 만드는 기존안은 원격 개인정보 처리의 법적 요건을 충족했다고 입증할 수 없어 선택하지 않는다.
- 모든 방문자에게 먼저 보호자 로그인을 요구하면 게스트 학습 원칙을 깨므로 선택하지 않는다.
- 원격 기능 전체를 영구 포기하면 기기 변경 복구 목표를 달성할 수 없어, 기본 비활성·단계적 승인을 택한다.

## 결과

Phase 5A의 로컬 병합·백업·mock transport는 진행할 수 있다. 공개 원격 생성, 로그인, 복구, 보호자 연결은 개인정보 검토와 운영 정책 승인을 별도 Gate 5C로 통과해야 한다. 결정 0003의 “원격 계정 생성에 보호자 승인이 필요하지 않다”는 부분만 이 결정이 대체한다.
