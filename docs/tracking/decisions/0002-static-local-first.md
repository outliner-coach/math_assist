# 정적 배포와 기기 로컬 기본 저장

## 상황

이 프로젝트는 비상업적 개인 교육 앱이며, 어린이가 계정 설정 없이 바로 문제를 풀 수 있어야 한다. 현재 콘텐츠와 채점은 서버 계산 없이 브라우저에서 수행할 수 있다.

## 결정

앱은 GitHub Pages에 정적 파일로 배포하고 교육 콘텐츠도 저장소 안의 정적 데이터로 관리한다. 학습 세션과 진도는 localStorage를 기본 저장소로 사용한다. 원격 저장이 추가되어도 로컬 저장을 제거하지 않고 같은 저장 인터페이스의 선택적 두 번째 구현으로 둔다.

## 선택하지 않은 안

- 처음부터 데이터베이스와 관리자 화면을 두는 방식은 개인 프로젝트 규모에 비해 운영·마이그레이션 부담이 크고 콘텐츠 검토를 코드 변경과 분리하므로 선택하지 않았다.
- 서버 세션만 사용해 온라인 상태에서만 학습하게 하는 방식은 로그인 없는 즉시 시작과 오프라인에 가까운 복구를 깨뜨려 선택하지 않았다.

## 결과

정적 경로는 빌드 때 모두 생성되어야 하며 런타임 API를 가정할 수 없다. 콘텐츠 수정은 저장소 변경과 배포가 필요하다. 브라우저 데이터 삭제나 기기 변경 시 기록이 사라지는 한계가 남으며, 선택적 로그인은 이 한계를 보완하되 학습 진입 장벽이 될 수 없다.
