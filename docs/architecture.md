# 시스템 구성

Math Assist는 한국 초등학생이 가입 없이 바로 시작할 수 있는 정적 웹 학습 앱이다. 현재 작업트리는 1·2·3·4학년의 미션형 학습, 5학년의 개념별 10문제 연습, 6학년 비와 비율의 5/10문제 Study를 제공한다. 4·6학년은 검증된 한 단원씩만 공개하며 단일 public-ledger gate가 미승격·원장 오류 상태의 직접 학습 경로를 닫는다.

## 구성과 의존 방향

```text
학습자 브라우저
  ├─ 화면과 경로(src/app)
  │    ├─ 공용 표현 컴포넌트(src/components)
  │    └─ 학습 규칙·문제 생성·채점·저장 어댑터(src/lib)
  ├─ 정적 교육 콘텐츠(public/data)
  └─ 기기 로컬 기록(localStorage)

검증 스크립트(scripts) ──읽기──> 교육 콘텐츠와 생성 규칙
GitHub Actions ──빌드──> 정적 산출물(out) ──배포──> GitHub Pages
```

- `src/app`은 경로 진입, 화면별 상태 조립, 사용자 행동 연결을 소유한다. 수학 정답 계산과 영속 형식의 정규화는 소유하지 않고 `src/lib`에 의존한다.
- `src/components`는 문제·도형·키패드·풀이장·보상 표현을 소유한다. 원천 문제를 다시 계산하거나 진도 정책을 결정하지 않는다.
- `src/lib`는 문제 은행, 결정적 생성과 채점, 진도·세션 정규화, 학습 홈의 여러 저장 형식 읽기를 소유한다. 화면 구조에는 의존하지 않는다.
- `public/data`는 5·6학년 단원·개념·문제 템플릿의 원천이다. Grade 6 ID와 파일 접두사는 Grade 5와 겹치지 않으며 앱 코드는 이 데이터를 읽지만 데이터가 앱 코드를 참조하지 않는다.
- `src/lib/grade-release.ts`는 공개 curriculum ledger의 학년별 release state를 읽는 단일 fail-closed 경계다. Grade 6 index와 공유 unit·concept·practice·result 화면은 이 경계를 통과하기 전 저장 repository를 호출하지 않는다.
- `scripts`는 콘텐츠를 제품과 같은 규칙으로 검사한다. 검증 결과를 런타임 상태로 사용하지 않는다.
- GitHub Pages는 정적 파일을 제공하고, 브라우저 번들에 포함된 KaTeX가 개념·문제·풀이의 수식을 HTML로 렌더링한다. KaTeX는 원격 서비스나 CDN 호출이 아니다. 현재 서버, 데이터베이스, 인증 제공자, 원격 AI 호출은 없다.

## 대표 흐름: 5학년 개념 연습

1. 정적 빌드는 단원과 개념 식별자를 읽어 `/unit/[unitId]`, `/concept/[conceptId]`, `/practice/[conceptId]` 경로를 미리 생성한다.
2. 브라우저는 `NEXT_PUBLIC_BASE_PATH`가 포함된 `/data` JSON을 가져와 개념과 선택한 A·B·C 세트의 템플릿을 읽는다.
3. `src/lib`가 한 세트에서 난이도 1·2·3을 4·4·2로 뽑고, 같은 매개변수로 문장·정답·풀이·시각 데이터를 만든다.
4. 화면은 한 문제씩 보여 주고 답을 로컬 세션에 저장한다. 답을 확인하면 규칙 기반 채점 결과와 풀이가 즉시 공개되며 확인한 답은 바꿀 수 없다.
5. 모든 문제를 확인하면 결과와 개념별 진도를 로컬에 저장하고 활성 세션을 지운 뒤 `/result`로 이동한다.
6. 오답 재도전은 직전 결과의 틀린 문제 스냅샷만 새 세션으로 만들므로 템플릿 변경이 진행 중 복습의 정답을 바꾸지 않는다.

## 대표 흐름: 1·2·3학년 미션

1. 학년별 화면은 코드에 포함된 단원·미션 은행에서 선택한 미션의 구체 문제를 결정적 시드로 만든다.
2. 답안 형식에 맞는 입력을 정규화하고 규칙 기반으로 정오를 판정한다. 형식 오류는 학습 오답과 구분한다.
3. 형식 검사를 통과해 정오가 확정된 확인은 `AttemptReceipt` 추가 원장에도 멱등하게 기록한다. 재시도 순번은 의도적인 다음 확인을 구분하고, 같은 확인 동작의 반복 전달은 중복으로 처리한다. 원답 문자열과 풀이장 획은 receipt에 넣지 않는다.
4. 풀이장은 학년별 결정적 실행 키와 구체 문제 variant로 만든 `sessionId + itemId`에 정규화 획을 자동 저장한다. 문항 이동·새로고침은 같은 문서를 복구하고 명시적 재시작은 새 세션을 사용한다. 확인 완료·만료 문서는 읽기 전용이다.
5. 성공한 구체 문제는 기존 학년별 진행 기록에 반영된다. 이 기록이 보상·복습의 권위이며, 1·2학년은 문제 내용 지문으로 중복 보상을 막고 학습일·복습·숙련·경험치를 파생한다.
6. 홈은 학년별 저장 형식을 수정하지 않고 읽어 이어하기 한 가지 행동으로 투영한다.

## 저장 경계의 목표 상태

현재 화면은 다섯 개 이상의 localStorage 이름을 직접 또는 학년별 도우미를 통해 사용한다. 계정 저장을 추가하기 전에 학습 세션, 진도, 풀이장, 학습자 프로필을 각각 저장소 인터페이스 뒤로 옮기고 기존 키를 무손실로 읽는 어댑터를 둔다. 로컬 구현이 기본이며 원격 저장은 두 번째 구현이다. 미션·캐릭터·보상은 이 저장 경계를 소비할 뿐 문제 생성·채점·복구를 통제하지 않는다.

로그인 없이 시작한 기록은 `기록 저장하기`를 선택할 때 생성되는 계정으로 연결한다. 첫 원격 동기화 범위는 완료·복습·최근 활동·아바타이며 풀이장 획과 화면 애니메이션 상태는 포함하지 않는다.
