01 · Diagnosis

차이는 문제가 아닙니다.
단절이 문제입니다.

현재 Math Assist에는 서로 보완적인 강점이 있습니다. 1·2학년의 미션·지도·보상은 학습을 시작하게 만들고, 5학년의 풀이장은 생각의 과정을 밖으로 꺼내 줍니다. 이제 두 강점을 전 학년에 적용하되 1–2, 3–4, 5–6학년에서 세션 단위와 기능 밀도를 다르게 조절해야 합니다.

GAP 01 · CONTENT

학년별 완성도가
서로 다른 단계

1·2학년은 V1 규모, 3학년은 Alpha, 5학년은 강한 템플릿 은행입니다. 4·6학년은 아직 비어 있어 “1–6학년 서비스” 약속을 할 수 없습니다.

해법: 공식 교육과정 추적성 + 학년별 최소 콘텐츠 기준 + 동일 품질 게이트
GAP 02 · IDENTITY

기록은 있지만
학습자는 없습니다

진도·복습·XP는 기기에 저장되지만 학습자 프로필에 귀속되지 않습니다. 한 기기를 형제자매가 공유하거나 기기를 바꾸면 기록의 연속성이 깨집니다.

해법: 게스트 로컬 기록 유지 + 동의가 확인된 선택적 원격 계정 + 무손실 이관
GAP 03 · EXPERIENCE

시각 언어보다
행동 규칙이 다릅니다

풀이장은 5학년에만, 강한 미션·보상은 저학년에만 있습니다. 서로 좋은 요소가 학년 경계에 갇혀 있어 아이가 학년을 올라갈 때 제품 경험이 끊깁니다.

해법: 공통 풀이장·미션·캐릭터 엔진을 만들고 표현 강도만 연령별 적응

통일의 단위는 ‘학습 루프’

모든 학년이 미션을 받고, 풀이장으로 생각하고, 피드백을 받고, 캐릭터와 함께 숙달을 쌓습니다. 지도 표현, 보상 톤, 풀이장 도구 수, 세션 길이만 학년군에 맞게 달라집니다.

06 · Identity & login

아이에게 로그인시키지 말고,
기록과 아바타를 연결합니다.

학습 시작은 계속 게스트로 열어둡니다. 어린이는 “기록 저장하기”를 직접 선택하고 이메일 대신 자동 생성 학습 번호와 4자리 PIN으로 돌아옵니다. 다만 만 14세 미만 기록을 서버에 만들고 동기화하는 단계는 법정대리인 동의와 확인을 통과해야 합니다. 첫 동기화 범위는 완료·복습·최근 활동·선택 아바타로 제한하고 풀이 획은 필요가 확인된 뒤 연결합니다.

Design rule
No login wall.
Never lose progress.
  • 게스트는 이름·이메일 없이 바로 첫 문제를 풉니다.
  • 어린이가 기록 저장을 선택하면 학습 번호·4자리 PIN·일회성 복구 카드를 준비합니다.
  • 원격 계정 생성 전 법정대리인 동의와 동의자 확인을 완료합니다.
  • 보호자 연결은 동의·복구·여러 어린이 프로필 관리에만 사용하고 학습 시작을 막지 않습니다.
  • 연결 시 기존 localStorage 기록을 미리 보여 주고 서버 확인 뒤에도 로컬 백업을 유지합니다.
  • 삭제·보존·연결 해제·세션 만료 정책과 개인정보 검토가 끝나기 전에는 원격 기능을 공개하지 않습니다.
