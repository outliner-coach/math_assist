# 5학년 교육 콘텐츠 작업 규칙

## 담당 범위

이 디렉터리는 5학년 단원, 개념 설명, A·B·C 문제 템플릿과 공개 정적 자산을 담당한다. 생성 알고리즘, 답 파서, 화면 레이아웃, 1·2·3학년 코드 기반 미션 은행은 담당하지 않는다.

## 경계

- `units.json`의 식별자는 정적 경로 입력이고 `concepts.json`의 `unit_id`는 그 식별자와 일치해야 한다.
- 템플릿 파일 이름은 개념 식별자의 첫 `-` 앞부분과 일치해야 하며 각 항목의 `concept_id`가 실제 개념을 가리켜야 한다.
- 새 필드나 새 시각 모양은 JSON만 먼저 추가하지 않는다. 공유 타입, 생성기, 렌더러, 검증기와 집중 테스트가 같은 변경에서 이해해야 한다.
- 교육과정 근거와 외부 조사는 이 디렉터리의 JSON 주석처럼 남기지 않고 별도 근거 파일에 보존한다.

## 항상 유지할 동작

- 모든 템플릿은 `set_id` A·B·C, 난이도 1·2·3, 매개변수 범위, 문제 문장, 정답 규칙, 풀이 단계를 가진다.
- 개념별 각 세트는 10개이며 난이도 분포는 4·4·2다.
- 객관식은 서로 다른 보기 네 개와 정확히 한 정답을 모든 허용 매개변수 조합에서 보장한다. 구조적으로 보장할 수 없는 “아닌 것은” 유형을 쓰지 않는다.
- 매개변수에 0·1이나 겹치는 범위가 들어가도 정답·보기·분모·도형이 유효해야 한다. 보장할 수 없으면 범위를 좁힌다.
- 문제 문장, 정답, 풀이, 그림은 같은 매개변수에서 계산한다. 답 전용 시각값을 문제 데이터처럼 공개하지 않는다.
- A·B·C는 숫자만 바꾸지 않고 맥락·표현·추론 방식 중 하나가 달라야 한다.

## 표현식과 문제 설계

- 등록된 함수와 단순 산술만 사용한다. 삼항 연산, 비교 연산, 메서드 체인, 중첩 함수가 필요하면 먼저 생성기 계약을 확장한다.
- 오답은 정답에 서로 다른 안전한 오프셋을 두거나 수학적으로 불가능함이 증명되는 구조를 사용한다. 서로 다른 함수 결과가 우연히 같지 않을 것이라고 가정하지 않는다.
- 힌트는 2~3개의 핵심 단계로 답 자체보다 사고 순서를 제공한다. 풀이 단계는 단위와 중간 계산을 포함해 정답 규칙과 일치해야 한다.
- 정량 시각은 하나의 수학 모델에서 위상·비율·라벨을 생성할 수 없는 상태로 추가하지 않는다.

## 테스트

- 변경한 JSON은 `npm run validate:templates`와 `npm run audit:problems`를 통과해야 한다.
- 새 수학 함수나 표현식은 생성기 집중 테스트에서 성공·실패 경계를 검사한다.
- 답 노출이나 도형 의미가 바뀌면 렌더러 집중 테스트와 브라우저 테스트를 추가한다.
- 템플릿 변경 검증은 기존 활성 세션을 지우고 새 세션에서 수행한다. 이미 저장된 문제 스냅샷이 바뀔 것으로 기대하지 않는다.
