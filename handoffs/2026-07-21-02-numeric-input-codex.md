# Phase 1 - 5학년 숫자 입력 형식 경계

## 변경

- `normalizeAnswer`가 `parseFloat`의 부분 파싱에 기대지 않고 지원하는 소수 형식 전체와 일치할 때만 정규화한다.
- `1/`, `-`, `.`, `-.`, `1.`, `1 1`, `1 1/`처럼 완성되지 않은 분수·소수·대분수와 뒤에 문자가 붙은 입력을 형식 오류로 구분한다.
- 연습 화면은 구체적인 형식 안내를 표시하고 `checkedAnswers`를 `null`로 유지한다. 입력을 고치면 안내가 사라지고 같은 문제를 계속 확인할 수 있다.
- 음수 대분수 `-0 1/2`의 부호도 입력 문자열의 부호를 기준으로 보존한다.

## 검증 계약

- 단위 테스트는 미완성 입력 거부, 완성된 정수·소수·분수·대분수 허용, 구체적 안내 문구를 고정한다.
- 브라우저 테스트는 `1/`, `-`, `.`, `1 1/`를 차례로 확인해도 피드백·답 잠금·오답 확인이 발생하지 않고, `1 1/2`로 완성한 뒤 정답 처리되는 흐름을 고정한다.

## 유지보수 주의

- 새 숫자 표기법을 추가할 때는 파서가 문자열 일부만 읽지 않도록 전체 일치 정규식을 먼저 추가한다.
- 형식 오류에서는 `markAnswerChecked`를 호출하지 않는다. 오답은 정규화에 성공한 수학값이 정답과 다를 때만 기록한다.
